# StartDesktop

GitHub Codespaces などの開発環境で仮想デスクトップ環境を起動するためのシェルスクリプトです。

## スクリプトの概要
このリポジトリには、以下の主要なシェルスクリプトが含まれています。

### デスクトップ環境
- **`start-xfce4.sh`**: XFCE4 デスクトップ環境を起動します。
- **`start-lxqt.sh`**: 軽量な LXQt デスクトップ環境を起動します。リソース消費を抑えたい場合に適しています。
- **`start-lxde.sh`**: 軽量な LXDE デスクトップ環境を起動します。

これらのスクリプトは、Xvfb で仮想ディスプレイを準備し、VNCサーバーとnoVNCを介してブラウザからデスクトップにアクセスできるようにします。

### ツールとユーティリティ
- **`run-qutebrowser.sh`**: キーボード中心のWebブラウザである qutebrowser をインストールして起動します。
- **`install-gemini.sh`**: Gemini CLI をインストールします。
- **`install_and_run.sh`**: 主要なスクリプトの実行権限を設定し、`install-gemini.sh`の実行と`start-lxqt.sh`によるデスクトップ環境の起動を自動で行うためのスクリプトです。

## 使い方

1.  GitHub Codespaces などのLinuxベースの開発環境で、このリポジトリをクローンまたはダウンロードします。
2.  ターミナルを開き、スクリプトがあるディレクトリに移動します。

### 推奨: `install_and_run.sh` の利用

基本的なセットアップとデスクトップ環境の起動を一度に行うには、このスクリプトを使用します。

1.  以下のコマンドでスクリプトに実行権限を付与します。
    ```bash
    chmod +x install_and_run.sh
    ```
2.  スクリプトを実行します。(初回は必要なパッケージをインストールするため、時間がかかります)
    ```bash
    ./install_and_run.sh
    ```
3.  スクリプトが正常に起動すると、Codespacesのポートフォワーディング機能により、ブラウザでアクセスするためのURLが表示されます。表示されたURLにアクセスしてください。

### 個別スクリプトの利用

特定のデスクトップ環境を手動で起動する場合は、各スクリプトに実行権限を付与して実行してください。

```bash
# 例: XFCE4を起動する場合
chmod +x start-xfce4.sh
./start-xfce4.sh
```

作業が終了したら、スクリプトを実行したターミナルで `Ctrl+C` を押して、全てのプロセスを終了させてください。
